## Opening URLs in the user's browser

When the user asks to "open", "go to", "pull up", or "navigate to" a URL/website WITHOUT also asking you to scrape, click, fill, summarize, or interact with the page — use Bash with SSH, NOT the MCP chrome tools.

The MCP chrome tools (mcp__claude-in-chrome__*) open isolated sandbox windows. Bash+SSH opens the URL as a new tab in the user's actual Chrome, merged with their existing browsing session — which is what they want for "just open this for me" requests.

Target host selection (configure GAMEZ_SSH_HOST, GAMEZ_SSH_USER, MAC_SSH_HOST, MAC_SSH_USER in .env):
- Default (no host mentioned, or user says "gamez", "my laptop here", "this laptop"):
  `ssh $GAMEZ_SSH_USER@$GAMEZ_SSH_HOST 'DISPLAY=:0 google-chrome "<URL>" >/dev/null 2>&1 &'`
- Mac (user says "on Mac", "on my Mac", "on the Mac"):
  `ssh $MAC_SSH_USER@$MAC_SSH_HOST 'open -a "Google Chrome" "<URL>"'`

Quote the URL with double quotes to handle ampersands and other shell metacharacters.

After the shell command returns (even with `&`), tell the user one line: "Opened on gamez." or "Opened on Mac." If they asked for something interactive on top of opening (e.g. "open this and tell me what's on the page"), also use the MCP tools after the Bash open to read/interact, or skip Bash and use MCP tools end-to-end.

Only use the MCP chrome tab tools when the user explicitly wants Jarvis to DO something with the page — click, fill, scrape, summarize content, extract data. For "just open it for me", always use Bash+SSH.
